# ScIGMA Project Architecture and Development Guide

## Overview
**ScIGMA** (Single-cell Inference of Genotypes and Multi-omics Analysis) is an interactive R/Shiny application and Bioconductor package designed to analyze, visualize, and integrate single-cell DNA and protein (scADT-seq) data, specifically generated by the Tapestri platform.

This document serves as a reference for the architecture, dependencies, and contribution guidelines, specifically targeting BioConductor compliance.

## System Architecture

ScIGMA follows a modular Shiny architecture built on top of the `golem` framework. 
State management is handled via R6 (`utils_ScIGMA_R6.R`), allowing the persistence of the `MultiAssayExperiment` (MAE) across different analysis modules.

### Core Modules
1. **H5Processing**: Parsers for `.h5` files from Tapestri outputs. Uses `DelayedArray` and `HDF5Array` for out-of-core computing to limit memory usage.
2. **DNA Variant Filtering**: Removes noisy variants based on quality metrics, VAF, and DP.
3. **CNV Processing**: Infers copy number alterations and maps amplicons to specific genomic regions.
4. **COMPASS Inference**: Runs an embedded C++ MCMC engine via Rcpp to infer the phylogenetic tree of clones from sparse/noisy genotype matrices.
5. **Protein Analysis**: Integrates DSB normalization, dimensionality reduction (PCA, UMAP), and unsupervised clustering.
6. **Multiomics Integration**: Cross-analyzes the inferred DNA clones against the protein sub-populations.

## Bioconductor Compliance

To ensure `ScIGMA` is ready for a Bioconductor submission, the following strict rules are implemented:
- **Zero `library()` or `require()` calls**: All dependencies must be formally declared in the `DESCRIPTION` and called via `pkg::function()` or `importFrom`.
- **Memory Optimization**: Must not load full count matrices into memory unless explicitly subsetted.
- **Strict Typing**: Usage of `vapply()` is enforced over `sapply()` to guarantee deterministic return types.
- **Seed Management**: Randomness via `set.seed()` is forbidden inside the package functions and is strictly reserved for the testing suite.
- **Testing**: A comprehensive suite of unit tests (`testthat`) is required, providing >80% code coverage.

## Running Tests
Run the entire suite locally before any commits:
```r
devtools::test()
```

For a rigorous Bioconductor compliance check:
```r
BiocCheck::BiocCheck(".")
```
